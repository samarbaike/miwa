What we are building
Miwa is a real-time competitive 1v1 quiz app. Players compete in ranked matches answering Math, Grammar, and Analogy questions. The system supports human-vs-human matchmaking with an ELO skill window, human-vs-Ghost-bot matches, and private invite lobbies. Built in Python using FastAPI, PostgreSQL, and WebSockets, structured entirely with OOP principles for your coursework submission.

FeatureDescriptionAuthRegister and login with bcrypt hashing. Session stored in the database.MatchmakingELO window starts at +-15, expands +-5 every 10 seconds until a match is found.Bot opponentAssigned if no human found. Ghost bot replays exact answers and reaction times of a historical top-scoring player for each specific question.Real-time game8-event WebSocket protocol drives the full game from connect to final result.Scoring1 point per correct answer. Speed is tie-breaker. ELO updates after every match.Match modesRanked (ELO changes), Practice (no ELO), Invite (private room code).Questions3 categories: Math, Grammar, Analogy. 4-option multiple choice. Optional image. Entered directly via psql.LeaderboardTop players sorted by ELO rank.ProfilePlayer stats: rank, wins, streak, total matches, match history.Frontend6 pages: Login, Home, Matchmaking, QuizBattle, Leaderboard, Profile.
Project folder structure
Set this up on Day 1 (done) and never reorganise it. Routes call services. Services contain all logic. Models define the database. Schemas define what enters and exits the API. There is no data/ folder and no scripts/ folder.

File / FolderPurposeapp/main.pyEntry point. Creates the FastAPI app and mounts all routers.app/database.pyDatabase connection. Shared across the whole app.app/models/SQLAlchemy classes that define database tables. One file per entity.app/schemas/Pydantic classes defining what data looks like entering and leaving the API.app/services/All business logic. Routes call these. Services call models.app/routes/FastAPI routers. Thin -- receive request, call service, return response.app/websocket_manager.pyConnectionManager class. Tracks all active WebSocket connections.app/core/events.pyClass containing all WebSocket event name constants.app/core/security.pyPassword hashing and verification helpers.frontend/HTML, CSS, JS for all 6 pages. Vanilla only -- no framework.docs/Project documentation: testing checklist, WebSocket protocol, API table.
