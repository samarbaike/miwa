What we are building
FeatureDescriptionAuthRegister and login with bcrypt hashing. Session stored in the database.MatchmakingELO window starts +-15, expands +-5 every 10 seconds until a match is found.Ghost botAssigned if no human found. Replays exact answers and reaction times of a historical top-scoring player per question. Data seeded by a separate script later.Real-time game9-event WebSocket protocol drives the full match from connect to final result.Scoring1 point per correct answer. Speed is tie-breaker. ELO updates after every match.Match modesRanked (ELO changes), Practice (no ELO), Invite (private room code).Questions3 categories: Math, Grammar, Analogy. 4-option multiple choice. Optional image. Entered via psql directly.LeaderboardTop players sorted by ELO rank.ProfileRank, wins, streak, total matches, match history.Frontend6 pages: Login, Home, Matchmaking, QuizBattle, Leaderboard, Profile.
Project folder structure
File / FolderPurposeapp/main.pyEntry point. Creates the FastAPI app and mounts all routers.app/database.pyDatabase connection. Shared across the whole app.app/models/SQLAlchemy classes defining database tables. One file per entity.app/schemas/Pydantic classes defining what data looks like entering and leaving the API.app/services/All business logic. Routes call these. Services call models.app/routes/FastAPI routers. Thin -- receive request, call service, return response.app/websocket_manager.pyConnectionManager class. Tracks all active WebSocket connections.app/core/events.pyAll WebSocket event name constants as class attributes.app/core/security.pyPassword hashing and verification helpers.frontend/HTML, CSS, JS for all 6 pages. Vanilla only.docs/Testing checklist, WebSocket protocol reference, API table.
